Tesseract

A CSS-Framework on Principles

Principles

In software development, principles are used to make complex things managable

Single responsibility

A class should have responsibility over only a single part of your system 

.padding-all-medium {
  padding: 15px;
}

.margin-bottom-large {
  margin-bottom: 20px;
}

.border-all {
  border: 1px solid transparent;
}

.border-radius-small {
  border-radius: 4px;
}

Single responsibility 

.alert {
  padding: 15px;
  margin-bottom: 20px;
  border: 1px solid transparent;
  border-radius: 4px;
}

Responsible for multiple things

Separation of concerns

Do not mix concerns by styling native html elements that you may want to use just for their semantic reason 

<h1 class="headline-large">Thats my Seo optimized title</h1>
<h3 class="headline-large">Thats another realy big text, visually</h3>

.headline-extra-large {
  @extend .font-size-large
  @extend .font-weight-bold
  @extend .margin-bottom-small
}

Concerns seperated 

<h1>Thats my Seo optimized title</h1>
<h1>Text that should look like h1</h1>

h1 {
  font-size: 6em;
  font-weight: bold;
  margin-bottom: 16px;
}

Concerns not seperated

Conceptual integrity

Stick to a single, simple set of design principles and stay consistent 

.font-size-small {
  font-size: 100%;
}

.font-size-medium {
  font-size: 110%;
}

.padding-left-large {
  padding-left: 10%;
}

Consistent 

.font-size-small {
  font-size: 100%;
}

.font-size-medium {
  font-size: 4em;
}

.padding-left-large {
  padding-left: 80px;
}

Inconsistent

Economic

Prevent unnecessary complexity. Focus on what is really needed and limit yourself 

.font-size-small {
  font-size: 100%;
}

.font-size-medium {
  font-size: 110%;
}

.padding-left-large {
  font-size: 140%;
}

Limited complexity 

.font-size-xxs {
  font-size: 70%;
}

.font-size-xs {
  font-size: 90%;
}

.font-size-s {
  font-size: 100%;
}

.font-size-m {
  font-size: 110%;
}

.font-size-l {
  font-size: 140%;
}

.font-size-xl {
  font-size: 160%;
}

.font-size-xxl {
  font-size: 200%;
}

Unfocused, useless complexity

Visibility & symmetry

Make classnames predictable by keep a consistent and comprehensible way of terminology 

.font-size-small {
  font-size: 100%;
}

.margin-top-medium {
  margin-top: 5%;
}

.padding-bottom-large {
  padding-bottom: 10%;
}

Comprehensible, predictable wording 

.fsS {
  font-size: 100%;
}

.mt-M {
  margin-top: 5%;
}

.pbL {
  padding-bottom: 10%;
}

Incomprehensible wording

Self-organized

Scaleable, efficent and economical system with self-organized, decentralized components 

.table {
  display: table;
}

.table-cell {
  display: table-row;
}

.table-row {
  display: table-cell;
}

Self-organized 

.table {
  display: table;
}

.table .table-row {
  display: table-row;
}

.table .table-row .table-cell {
  display: table-cell;
}

Dependent, not reusable

Abstraction, Structuring, Hierarchising, Modularity

Abstract, structure, hierarchise repeating patterns to make them reusable/modular 

.border-top {
  border-top: 1px solid grey;
}
.border-right {
  border-right: 1px solid grey;
}
.border-bottom {
  border-bottom: 1px solid grey;
}
.border-left {
  border-left: 1px solid grey;
}
.border-all {
  @extend .border-top;
  @extend .border-right;
  @extend .border-bottom;
  @extend .border-left;
}

Hierarchised, abstracted, modularized 

.border-top {
  border-top: 1px solid grey;
}
.border-right {
  border-right: 1px solid grey;
}
.border-bottom {
  border-bottom: 1px solid grey;
}
.border-left {
  border-left: 1px solid grey;
}
.border-all {
  border-top: 1px solid grey;
  border-right: 1px solid grey;
  border-bottom: 1px solid grey;
  border-left: 1px solid grey;
}

Repeated, redundant
